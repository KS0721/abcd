# 올인원 AAC - AI/LLM 통합 설계서

> 리스크 분석을 통해 8개 중 3개 기능만 선별하여 적용합니다.
> 대상 사용자가 약자(장애인)이므로 안전성과 혼란 방지를 최우선으로 합니다.

---

## 핵심 원칙

- **0 추가 터치**: AI는 백그라운드에서 작동, 사용자 조작 불필요
- **폴백 보장**: LLM 실패 시 기존 규칙 기반 동작 100% 유지
- **개인정보 보호**: 모든 데이터 로컬 처리 우선
- **지연시간 최소화**: 500ms 이내 응답

---

## 적용 기능 3가지

### 1. 한국어 자동 문법 향상 (grammarService.ts)

**현재**: 규칙 기반 자동 조사 (받침 판별 + 카테고리 분석, 커버리지 ~80-90%)
**LLM 확장**: 문맥 기반 조사 선택 + 높임말 변환 + 연결어미 생성

| 케이스 | 규칙 한계 | LLM 해결 |
|--------|----------|----------|
| 은/는 vs 이/가 | 주제 vs 주어 구분 불가 | 대화 문맥으로 판단 |
| 에게 vs 한테 | 구분 어려움 | 구어/문어 문맥 판단 |
| 높임말 | 드세요/주세요 변환 | 상대방에 따라 자동 |

### 2. 스마트 카드 추천 (SuggestionBar.tsx)

**현재**: 정적 매핑 (동사 → 관련 명사 추천)
**LLM 확장**: 선택 카드 + 시간대 + 발화 이력을 분석하여 다음 카드 5~8개 추천

### 3. 기록 기반 패턴 학습 (HistoryScreen.tsx)

**현재**: 발화 기록 단순 목록 표시
**LLM 확장**: 빈도/시간대/연속 패턴 분석으로 "다음에 말할 문장" 자동 추천

## 제외된 기능과 사유

| 기능 | 제외 사유 |
|------|----------|
| 개인화 문장 예측 | 부적절한 추천 시 사용자 혼란/불쾌감 위험 |
| STT 상대방 음성 인식 | 프라이버시 침해 + 오인식 시 대화 흐름 방해 |
| 상황 자동 감지 | GPS 수집 부담 + 오감지 시 혼란 |
| 스마트 카드 정렬 | 카드 위치 변경 시 학습된 위치 기억 파괴 |
| 대화 흐름 총괄 제어 | 과도한 AI 개입으로 사용자 자율성 침해 |
